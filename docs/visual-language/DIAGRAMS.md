aiio separates illustrations from diagrams.

## BC-DIAGRAM-001

### Editorial Progression

Purpose:

Visualize conceptual progression.

Not process execution.

Not software architecture.

Not workflow.

### Typical Use

Information + AI

↓

Organizational Self-Understanding

↓

Coordinated Action

### Composition Rules

- No surrounding container.
- No filled boxes.
- No dashboard appearance.
- No UI cards.
- Use generous whitespace.
- Use elegant typography.
- Use subtle connecting lines.
- Use minimal arrows.
- Keep the progression readable without surrounding explanation.

### Spacing Rules

- Each concept needs enough space to stand as an idea.
- Connectors should be quiet and secondary.
- Whitespace creates authority.
- Diagrams should feel like figures in a premium management publication.

### Color Usage

- Graphite: current organizational reality.
- Cyan: understanding.
- Purple: Organizational Intelligence or capability generation.
- Amber: business outcome.

Color must reinforce meaning.

Color must never decorate.

### Prohibited Styles

- Interface cards.
- Presentation-slide boxes.
- Dashboard modules.
- Heavy arrows.
- Workflow notation.
- Decorative diagrams.
