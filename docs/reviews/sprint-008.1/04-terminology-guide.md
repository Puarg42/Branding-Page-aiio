# 04 - Terminology Guide

## Purpose

Define consistent terminology for the visible website copy.

| Term | Decision | Usage Rule |
| --- | --- | --- |
| Organizational Intelligence | Primary category term | Use for the category and system-level idea. Avoid overusing it where "understanding" or "capability" is more precise. |
| Understanding | Central missing layer | Use when describing the shift from fragmented knowledge to meaning and context. |
| Capability | Observable outcome | Use when describing what organizations become able to do repeatedly and reliably. |
| Knowledge | Raw material | Use when describing organizational memory, documentation, experience and fragmented information. |
| Platform | aiio website section and implementation surface | Use as navigation and page context, but avoid making it the lead idea in headlines. |
| Operating system | System metaphor | Use sparingly where the OIS is described as infrastructure, not as a software feature. |
| Self-Enabling Organization | Long-term conceptual outcome | Use only as a major conceptual conclusion, not as repeated explanatory copy. |
| OIS | Avoid in primary visible CTAs | Spell out or use "system" where the audience may not know the acronym yet. |
